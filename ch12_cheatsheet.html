full structure
TRY Complete try/except/else/finally
try:
    # Code that might raise an exception
    result = 10 / 0

except ZeroDivisionError:
    # Runs if ZeroDivisionError is raised
    print("Cannot divide by zero")

else:
    # Runs ONLY if NO exception occurred
    print("Success:", result)

finally:
    # ALWAYS runs — exception or not
    print("This always runs")
BlockWhen it runs
tryAlways — contains risky code
exceptOnly when exception is raised
elseOnly when NO exception occurs
finallyALWAYS — with or without exception
except patterns
EXCEPT Multiple Exception Types
# Catch specific exception
try:
    x = int("abc")
except ValueError:
    print("Invalid number")

# Catch multiple in one line
except (ValueError, TypeError):
    print("Value or type error")

# Multiple except blocks
try:
    result = 10 / int(x)
except ZeroDivisionError:
    print("Divide by zero")
except ValueError:
    print("Not a number")
except Exception as e:
    print(f"Unexpected: {e}")  # catch-all
Never use bare `except:` — catches everything including KeyboardInterrupt and SystemExit
AS Accessing exception details
try:
    result = 10 / 0
except ZeroDivisionError as e:
    print(type(e))      # <class 'ZeroDivisionError'>
    print(str(e))       # division by zero
    print(e.args)       # ('division by zero',)
else & finally
ELSE else — success path
try:
    num = int("42")
except ValueError:
    print("Bad input")
else:
    print("Got:", num)
# Got: 42
Use else for code that should run only on success — cleaner than putting it in try
FINALLY finally — always runs
try:
    f = open("data.txt")
    data = f.read()
except FileNotFoundError:
    print("Missing")
finally:
    f.close()  # always close
Use finally for cleanup — close files, DB connections, release locks
raise
RAISE Raising Exceptions
# Raise a built-in exception
def set_age(age):
    if age < 0:
        raise ValueError(f"Age cannot be negative: {age}")
    return age

# Re-raise caught exception
try:
    result = 10 / 0
except ZeroDivisionError:
    print("Logging error...")
    raise              # re-raises same exception
custom exceptions
CUSTOM Creating & Using Custom Exceptions
# Define custom exception
class InsufficientFundsError(Exception):
    def __init__(self, balance, amount):
        self.balance = balance
        self.amount  = amount
        super().__init__(
            f"Need {amount}, only have {balance}"
        )

# Raise it
def withdraw(balance, amount):
    if amount > balance:
        raise InsufficientFundsError(balance, amount)
    return balance - amount

# Catch it
try:
    withdraw(100, 200)
except InsufficientFundsError as e:
    print(e)            # Need 200, only have 100
    print(e.balance)    # 100
Always inherit from Exception (not BaseException) for custom exceptions

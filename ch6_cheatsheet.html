Chapter 6 — Quick Revision Cheat Sheet

basics
DICT Creating Dictionaries
person = {"name": "Alice", "age": 30, "city": "London"}
empty  = {}
empty  = dict()

# Using dict() constructor
person = dict(name="Alice", age=30)

print(type(person))   # <class 'dict'>
print(len(person))    # 3
Key: Dicts are ordered (Python 3.7+), mutable, and keys must be unique and hashable
create / read / update / delete
READ Accessing Values
person = {"name": "Alice", "age": 30}

person["name"]              # "Alice" — KeyError if key missing
person.get("name")          # "Alice" — safe, returns None if missing
person.get("salary", 0)     # 0 — default value if key missing
get() vs []: Use get() when key might not exist — avoids KeyError
ADD / UPDATE Adding & Updating
person = {"name": "Alice"}

person["age"] = 30           # add new key
person["name"] = "Bob"       # update existing key

# update() — merge another dict in
person.update({"city": "Paris", "age": 25})

# setdefault() — add only if key doesn't exist
person.setdefault("age", 99)  # won't overwrite existing age
DELETE Removing Keys
person = {"name": "Alice", "age": 30, "city": "Paris"}

del person["age"]          # removes key — KeyError if missing
person.pop("city")         # removes & returns value
person.pop("salary", 0)    # safe pop — returns 0 if missing
person.popitem()           # removes & returns last inserted (key,value)
person.clear()             # empties dict → {}
key methods
VIEWS keys() / values() / items()
person = {"name": "Alice", "age": 30}

person.keys()    # dict_keys(['name', 'age'])
person.values()  # dict_values(['Alice', 30])
person.items()   # dict_items([('name','Alice'),('age',30)])

# Convert to list
list(person.keys())    # ['name', 'age']
Key: keys(), values(), items() return view objects — they reflect live changes to the dict
CHECK Membership & Copying
person = {"name": "Alice", "age": 30}

"name" in person        # True  — checks KEYS only
"Alice" in person       # False — values not checked
"Alice" in person.values()  # True

copy = person.copy()    # shallow copy
`in` checks KEYS only by default — use .values() to check values
looping through dicts
LOOP All Loop Patterns
person = {"name": "Alice", "age": 30, "city": "Paris"}

# Loop over keys (default)
for key in person:
    print(key)

# Loop over values
for val in person.values():
    print(val)

# Loop over key-value pairs
for key, val in person.items():
    print(f"{key}: {val}")
dict comprehension
COMP Syntax & Examples
# Syntax: {key: value  for item in iterable  if condition}

squares  = {x: x**2 for x in range(5)}
# {0:0, 1:1, 2:4, 3:9, 4:16}

even_sq  = {x: x**2 for x in range(10) if x%2==0}

# Swap keys and values
d       = {"a": 1, "b": 2}
flipped = {v: k for k, v in d.items()}
# {1: "a", 2: "b"}
nested dictionaries
NESTED Access & Update
users = {
    "alice": {"age": 30, "city": "Paris"},
    "bob":   {"age": 25, "city": "London"}
}

users["alice"]["age"]          # 30
users["bob"]["city"] = "NYC"   # update nested value

# Safe access with get()
users.get("charlie", {}).get("age", "N/A")  # "N/A"
merging dictionaries
MERGE All Ways to Merge
a = {"x": 1, "y": 2}
b = {"y": 99, "z": 3}

# update() — modifies a in-place
a.update(b)                  # a = {"x":1, "y":99, "z":3}

# ** unpacking — new dict (Python 3.5+)
merged = {**a, **b}           # b's values overwrite a's

# | operator — new dict (Python 3.9+)
merged = a | b               # cleanest syntax

# |= operator — update in-place (Python 3.9+)
a |= b
Conflict rule: When keys overlap, the RIGHTMOST dict wins
